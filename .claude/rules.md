# ⛔ 절대 금지 규칙

Claude Code는 **모든 작업 전에 이 파일을 먼저 읽어야 합니다.**

## 🚫 절대 하면 안 되는 것

### 1. 파일 삭제 금지
- ❌ 어떤 .js, .html, .css 파일도 삭제 금지
- ❌ api/ 폴더의 파일 삭제 금지
- ✅ 허용: 임시 파일, 로그 파일만 삭제 가능

### 2. 비즈니스 로직 변경 금지
- ❌ API 엔드포인트 URL 변경 금지
- ❌ 결제 관련 코드 수정 금지
  - `api/checkout.js`
  - `api/webhook.js`
  - `upgrade.js`
  - LemonSqueezy 관련 모든 코드
- ❌ 사용량 추적 로직 변경 금지 (`usage.js`)

### 3. 환경변수 관련 금지
- ❌ .env 파일 삭제 금지
- ❌ .env 파일 내용 출력 금지
- ❌ .env 파일을 Git에 추가 금지
- ❌ 환경변수 값 변경 금지 (변수명 오타 수정은 OK)

### 4. 디자인 대폭 수정 금지
- ❌ style.css에서 주요 색상 변경 금지
- ❌ 레이아웃 구조 변경 금지
- ✅ 허용: 오타 수정, 미세한 여백 조정

### 5. 의존성 관련 금지
- ❌ package.json에서 의존성 삭제 금지
- ❌ npm uninstall 실행 금지
- ✅ 허용: npm install (새 패키지 추가)

### 6. Git 관련 주의
- ❌ `git push --force` 금지
- ❌ main 브랜치에서 직접 작업 금지
- ✅ 권장: 새 브랜치 만들어서 작업

## ✅ 해도 되는 것

### 안전한 작업들
- ✅ console.log 제거
- ✅ 주석 제거
- ✅ 사용하지 않는 import 제거
- ✅ 코드 포맷팅 개선
- ✅ 오타 수정
- ✅ JSDoc 주석 추가
- ✅ 보안 체크 및 수정
- ✅ 에러 수정 (로직 변경 없이)

2. [ ] 하려는 작업이 금지 목록에 없는가?
3. [ ] 백업(git commit)을 했는가?
4. [ ] 의심스러운 부분은 없는가?
